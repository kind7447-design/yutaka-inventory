# ユタカ製作所向け 支給品・調達品 在庫管理アプリ 設計書

最終更新: 2026-06-18

## 1. 目的・概要
客先（ユタカ製作所）からの**支給品**と、自社で手配する**調達品**を登録・在庫管理するWebアプリ。
固有番号＋QRコードで現場スマホから在庫確認、注文依頼書／加工依頼書の発行、入荷・使用履歴の記録までを行う。

## 2. 用語
| 用語 | 意味 |
|---|---|
| 支給品 | 客先から無償支給される部材（購入ではない／加工依頼書 不要） |
| 調達品 | 自社が外注先へ手配する購入品（加工依頼書 必要） |
| 消込 | 入荷予定に対し実入荷を確定し在庫を増やす操作 |
| 仕損 | 加工失敗等によるロス。使用報告に区分として持つ |

## 3. 技術構成（無料運用）
- バックエンド: **Python / FastAPI**
- DB: **PostgreSQL（Neon 無料プラン）** ※永続化のためRender無料の非永続ディスクは使わない
- ORM: SQLAlchemy ／ マイグレーション: Alembic
- フロント: サーバーサイドHTML（Jinja2）+ 最小JS。スマホ表示最適化（レスポンシブ）
- QR読取: スマホブラウザのカメラ（html5-qrcode等のJS）
- QR生成: Python `qrcode`（中身は item_code。読取→在庫確認画面へ遷移）
- PDF生成: `reportlab`（A5・注文依頼書／加工依頼書）
- ホスティング: **Render 無料Webサービス**
- 認証: 簡易ログイン（社内利用想定。ID/パスワード）
- **セキュリティ: 外部APIキーは使わない構成。使う場合もサーバー側のみ（クライアントに置かない）**

### 無料運用の制約と対策
- Render無料: 15分無アクセスでスリープ→次アクセス時 約30秒の復帰待ち（許容前提）
- Neon無料: 容量・接続数に上限あり。小規模利用なら十分

## 6. 帳票（PDF）
### 6.1 注文依頼書（A5・1枚に最大4案件）
- レイアウトは支給フォーマット（㈱林製作所書式）に準拠
- 見出し: 「注文依頼書」／伝票NO／日付
- 宛先: ◯◯ 殿 ／ 納入場所
- 明細: 材質・部品番号／種類／板厚／サイズ／数量／単位
- 備考: 納期／@単価／リピート
- footer: 自社名・住所・TEL/FAX
- A5用紙に4明細ブロックを配置

### 6.2 加工依頼書（調達品のみ）
- 図面＋寸法・材質・有効ネジ深さ等を表示
- 品目に添付した図面を貼り込み

### 6.3 QRコード印刷（A5）
- 品目マスタの「QRコード印刷」ボタンから出力
- 全件自動発行はしない。在庫管理が必要な品目だけを選んで必要時に印刷
- A5用紙を **2×2の4分割**。1区画＝QRコード＋下に製品番号(item_code)
- 面付け順：**左上→右上→左下→右下**。1つだけなら左上のみ（残りは空白）
- 5件以上選択時は4件ごとに次ページ
- QR内容＝item_code。スマホ読取で在庫確認画面へ

## 7. 在庫増減ロジック
- 受入(消込)確定 → receipt登録 → stock_qty += qty
- 使用/仕損報告 → usage_report登録 → stock_qty -= qty
- 整合性: 履歴合計＝在庫数 を検証できるようにする
